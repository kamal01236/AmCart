2. Functional Requirements (What the System Must Do)
2.1 User Management & Authentication

Users must be able to register and log in.

Users must be able to log in using social providers.

Users must be able to reset forgotten passwords.

The system must support role-based access (customer, admin).

Users must remain logged in using a “remember me” option.

2.2 Product Browsing & Discovery

Users must be able to browse products by category and subcategory.

Users must be able to view product details.

Users must be able to search products using keywords.

Users must be able to filter products by price, brand, color, rating.

Users must be able to sort products by price, popularity, and rating.

2.3 Cart & Checkout

Users must be able to add products to a cart.

Users must be able to update product quantities.

The system must calculate cart subtotal and order total.

Users must be able to apply coupon codes.

Users must be able to checkout as guest or logged-in user.

2.4 Orders & Payments

Users must be able to place orders.

The system must support multiple payment methods.

Orders must be created only after successful payment.

Users must receive order confirmation.

Users must be able to view order history.

2.5 Inventory Management

The system must track product stock levels.

Products that are out of stock must not be purchasable.

Users must be notified when out-of-stock items become available.

2.6 Reviews, Wishlist & Engagement

Users must be able to add products to a wishlist.

Users must be able to submit reviews and ratings.

Users must be able to view other users’ reviews.

The system must display testimonials and new products.

2.7 Administration

Administrators must be able to add, update, and delete products.

Administrators must be able to manage pricing and discounts.

Administrators must be able to manage inventory.

Administrators must be able to view out-of-stock products.

2.8 Notifications & Communication

Users must receive email notifications for key events.

The system must support a contact-us feature.

The system must maintain notification history.
